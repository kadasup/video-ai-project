---

## 1. 為什麼這類最該先自動化
「無觀點、平鋪直敘」的內容——社會案件敘述、股市/數據報讀、戰爭傷亡數字——**不需要導演創意**，只要忠實把文字/數據轉成有品牌感的短影音。
→ 高度標準化 = 最適合做成「**模板 ＋ 資料填充**」的產線；也正是會議說「車禍監視器這種沒觀點的、直接敘述就能出」的那一類。

## 2. 核心思路：別「每支從零剪」，要「文稿→成片」的模板產線
做幾套**固定的視覺模板**（片頭片尾、字型、配色、安全區、字幕樣式、配樂、logo 都固定），AI 只負責**填內容**（旁白稿、字幕、選畫面、數據卡）。編輯從「剪片」變成「**供稿＋審核**」。

```
新聞稿 / 數據（既有）
   │
① 腳本     LLM 產旁白稿＋分鏡＋標題/hook＋hashtag
② 配音     TTS 產旁白（無觀點數據類最適合）
③ 畫面     從影音庫(線3)找對應 B-roll / 監視器；無畫面數據類→自動資料卡
④ 字幕     依旁白自動對齊、燒錄（動態字幕）
⑤ 套模板   固定品牌包裝（依類型選模板）
⑥ 組裝     自動排時間軸 → 直接 render，或產 Premiere 專案供微調
⑦ 把關     編輯快速審核（事實/錯字/畫面）→ 出片
```

## 3. 三套「類型模板」（先各做一套）
| 類型 | 畫面來源 | 旁白 | 模板重點 |
|---|---|---|---|
| **社會案件**（車禍/監視器/無觀點） | 記者素材＋監視器畫面（影音庫搜） | TTS 播報 | 「今日社會」框、時間地點字卡、警示色 |
| **股市/數據** | 自動資料卡（漲跌/數字動畫），幾乎無實拍 | TTS 報讀數據 | 數據視覺化模板、表格→動畫 |
| **國際/戰爭報讀** | 地圖/示意/外電(⚠️版權) | TTS | 地圖定位、傷亡數字卡 |

## 4. 工具選型（給這個專案的建議）
**模板引擎（擇一或混用，由 AI 小組維護一次、編輯只填料）：**
- 🥇 **Remotion（用程式/React 生影片）**：最適合「數據卡、字幕、批次、資料驅動」，能真正全自動批量產出 → **股市/數據類首選**。
- 🥈 **After Effects 資料驅動模板（.aep + JSON/CSV）或 Premiere 動態圖形 .mogrt**：沿用既有 Adobe，編輯好微調 → **社會案件類**（要在 Premiere 收尾時）首選。
- ffmpeg / MoviePy：最輕量的程式化合成，適合超標準化、不要花俏的批次。

**旁白 TTS：**
- 預設用 **OpenAI TTS**（與既有金鑰一致）。
- 若要更像「台灣新聞主播」的自然口條，可 A/B 測 **Azure / Google 中文新聞語音**（台灣腔較成熟）。建議**固定一個品牌聲音**。

**選畫面：** 接 [[可行方案]] 線3 的影音庫語意搜片（社會案件類靠它找監視器/現場畫面）。

> 會議已明示「用 Codex / Python 寫程式 + 套模型」可接受，且會有 AI 窗口 → 程式化模板產線（Remotion/ffmpeg）由 AI 小組維護、編輯只供稿審核，這條路完全可行。

## 5. 編輯的新工作流（從「剪片」到「審片」）
1. 貼上新聞稿/數據 → 選類型模板。
2. 系統出：旁白稿草稿＋自動選好的畫面＋字幕＋成片草稿。
3. 編輯**審核微調**（改錯字、換不合適的畫面、調標題）。
4. 一鍵輸出（含多平台切版：YouTube 橫 / Reels・Shorts・IG・WeChat 直）。

> 守品質：事實、字幕錯字、畫面合適度 → 仍由編輯把關，AI 不直接無人出片。

## 6. 落地第一步（建議）
1. **拿這支毒駕範例當「基準模板」**：拆解它的結構（片頭、字幕樣式、配色、字體、片尾、配樂），做成第一套「社會案件」模板。
2. **做一支端到端 PoC**：拿一則新聞稿，跑「腳本→TTS→自動字幕→套模板→輸出」，和小編手做的這支比對：品質夠不夠、省多少時間。
3. **量成效**：手做 vs 產線各計時與品質評分 → 達標就批次套用到「被擋掉的舊社會案件素材」衝量。

## 7. 對準目標
- **省時**：把「每支從零剪 15 分鐘」變「填稿＋審核幾分鐘」。
- **增量**：模板化＋多平台切版 → 同一則可批量出多支、多平台。
- **品質不降**：包裝由模板保證一致、人工只把關內容，不靠 AI 亂剪。
