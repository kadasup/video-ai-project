## 0. 一頁總結（TL;DR）
- **目標**：用 AI 縮短影音產製時間、衝高產量，品質不降。
- **策略**：**先做兩個快速見效的（直播字幕、舊素材衝量），同時鋪基礎（影音庫搜片），最後做剪輯自動化。**
- **技術主軸**：原檔與剪輯留**地端**、AI 辨識上**雲端（OpenAI）**、直播字幕走**即時串流**。
- **硬體**：雲端標註免 GPU；但**地端剪輯/轉檔 GPU 仍要升級**（3060→5070 級）。
- **先動**：直播中文即時字幕 ＋ 舊素材模板化產出 ＋ 啟動搜片試點。

---

## 1. 目標與限制
| 目標 | 限制 |
|---|---|
| ① 縮短每日產製時間 | 品質不能降 |
| ② 大幅增加產片量 | 人力僅 3+1、預算/硬體待升級 |
| ③ 撐住未來爆量供稿 | 編輯半技術自管 → 要低維運、傻瓜化 |

### KPI 量測基準（數字待與主管拍板）
| 指標 | 現況基準 | 目標方向 |
|---|---|---|
| 每日產片量 | ~30 支（3+1 人） | 往選舉高峰 70~80 支區間拉近，或以「同人力翻倍」為目標 |
| 長轉短單支耗時 | ~15 分鐘（人工） | 顯著下降（改填稿＋審核制） |
| 找片時間 | 人工翻片 | 試點量測 AI vs 人工 |
| 品質 | — | 設字幕錯誤率、退件率上限 |
> 數字目標需與主管共同拍板；量測方法接 [[試點執行計畫]] 的成功指標。

## 2. 整體策略
**先「快速見效」拿成績、再「打基礎」、最後「自動化剪輯」。**
理由：會議要求先衝量、好交代；快速見效的題材技術獨立、風險低，可邊做邊建立合作模式與信心。

```
階段一(快速見效)     階段二(打基礎)        階段三(自動化剪輯)
─ 直播即時字幕        ─ 影音庫標註           ─ 腳本+選片配對(L2)
─ 舊素材模板化衝量    ─ 語意搜片(試點)        ─ 自動粗剪→Premiere(L3)
                                            ─ 長轉短/多平台切版(L4)
        並行          ───────────────►        依基礎成熟度接續
```

## 3. 四條工作線（做什麼／怎麼做／可行性）

### 線 1 ⭐ 直播即時字幕（先中文，後英文翻譯）
- **做什麼**：官網中／英文直播加上即時字幕（中文先），再加中→英翻譯字幕。
- **怎麼做**：直播音訊 → 串流語音辨識(streaming STT) → 即時字幕，用 **OBS 瀏覽器來源**疊到畫面；同步錄下供事後校對。維護一份新聞「熱詞表」提升人名/專名準確度。
- **可行性**：🟢 高（業界 TVBS／Apple 發表會已採；技術獨立、不動既有直播硬體）。
- **見效**：最快、最容易展示成果。

> 📎 **來源：[[研究彙整-國內外參考資料]]**
>
> **推薦 API：`gpt-realtime-whisper`（$0.017/min）**
> - WebSocket 端點：`wss://api.openai.com/v1/realtime?model=gpt-realtime-whisper`
> - 音訊格式：PCM16、**24kHz**（非 16kHz）、mono
> - VAD 建議用 `semantic_vad`（ML 斷句，比能量偵測自然）；語言提示加 `language: "zh"` 提升中文準確度
> - 已知問題：session 超過 9 分鐘延遲可能增至 6~8s → 每 8 分鐘重啟 session 作為 workaround
> - 實際延遲：獨立測試（webrtcHacks, 2025-01）為 1,660~1,860ms（非 OpenAI 官稱 232ms）
>
> **⚠️ `gpt-4o-transcribe` 不適合現場直播**
> 學術 benchmark（WenetSpeech）實測：乾淨錄音 CER 3.52% 看似良好，但真實會議/現場環境 CER **高達 31.40%**（每 100 字錯 31 個）。對照 Whisper large-v3 同場景 18.87%，差距顯著，新聞直播不可接受。
>
> **免費自架備援：WhisperLive**（PoC 測試或網路不穩時）
> ```bash
> pip install whisper-live
> python -m whisper_live.server --model large-v2 --language zh --port 9090
> ```
> GTX 1050Ti 可跑 small/medium 模型（large-v2 建議 6GB+ VRAM）。

### 線 2 ⭐ 模板化快速產製（衝量／KPI 亮點）
> 涵蓋多種「輸入是稿或數據、不靠觀點」的內容，共用同一套模板產線。完整類型見 [[可自動化內容類型盤點]]。
- **2a 數據型固定欄目（最該先做）**：天氣、地震速報、油價、匯率、賽事比分、財經盤後。輸入是官方/結構化數據、零編輯判斷 → 資料卡＋TTS，**可排程自動產出、無人值守**。可與財經/體育部門共用模板、統一品牌。
- **2b 無觀點社會案件**：車禍、監視器、道安等。AI 標註挑重點段 → 套模板（播報/TTS＋字幕＋標題）→ 批次輸出；正好消化過去被擋掉的舊素材（見 [[無觀點類短影音-快速產製建議]]）。
- **2c 官方記者會/政令、快訊速報**：長片自動切重點＋字幕／速報卡搶快。
- **可行性**：🟢 高（流程固定、無需導演創意；屬批次/排程處理）。
- **見效**：直接提升消化量；數據型欄目是「展示自動化成果」最漂亮的起手式。

> 📎 **來源：[[研究彙整-國內外參考資料]]**
>
> **TTS 選型（台灣中文）**
>
> | 方案 | 費用 | 台灣腔 | 建議 |
> |---|---|---|---|
> | **Azure zh-TW-HsiaoChenNeural** | 免費 50萬字/月，超額 $16/百萬字 | ⭐⭐⭐⭐⭐ | 正式上線首選；無兒化音、「和」字發音正確 |
> | **edge-tts（Python 套件）** | 完全免費 | ⭐⭐⭐⭐ | PoC 測試首選；與 Azure 同引擎，`pip install edge-tts` |
> | OpenAI TTS gpt-4o-mini-tts | $15/百萬字 | ⭐⭐⭐ | 腔調偏大陸普通話，不優先 |
> | Google Cloud TTS | $16/百萬字 | ⭐ | 台灣腔最差，不推薦 |
>
> 費用估算：30 支/天 × 每支 200 字 ≈ 18 萬字/月，低於 Azure 免費額度，**TTS 費用為零**。
>
> **模板影片工具**
> - **Remotion**（React/TS）：數據驅動動畫卡片首選（天氣/賽事比分），$25/dev/月
> - **ffmpeg `drawtext`**：純文字覆疊、字幕燒錄，免費
> - ❌ After Effects .mogrt：無法 Docker 化、需授權，不可用於自動化流程
>
> **最小垂直切片（建議最先 demo）**：
> 新聞稿 → GPT-4o mini 生腳本 → edge-tts → ffmpeg 合圖＋字幕 → mp4
> 純 Python + ffmpeg，無月費，1 週內可展示。

### 線 3 影音庫標註 ＋ 語意搜片（基礎建設・試點先行）
- **做什麼**：素材進庫自動標註，編輯打一句國語就找到片段。
- **含「自動預檢初篩」**：素材一進來，AI 先濾掉空拍／誤傳／糊晃片段、標記有新聞點的重點段供人審——把「看 45 分鐘海量素材」變成「審 AI 挑好的片段」（會議主管的核心願望）；篩選條件由人訂定，產出再餵給線4。
- **怎麼做**：本地切鏡頭/抽音訊/抽縮圖 → STT 逐字稿＋GPT-4o 看畫面＋標籤 → 向量庫 → 搜尋頁。**先用無隱私素材跑半個月試點**驗證效果與費用。
- **可行性**：🟢 高（方案已細化，見 [[試點執行計畫]]）。
- **見效**：是線 2、線 3、線 4 的共同地基。

> 📎 **來源：[[研究彙整-國內外參考資料]]**
>
> **STT 選型更新：優先考慮 FunASR（免費且更準）**
>
> | 模型 | AISHELL-1 CER | 會議/現場 CER | 費用 |
> |---|---|---|---|
> | **FunASR Paraformer-zh** | **1.64%** | ~5%（估） | 免費，本地自架 |
> | whisper-1（OpenAI API） | 5.14% | 18.87% | $0.006/min |
> | gpt-4o-transcribe | 3.52% | 31.40% ⚠️ | $0.006/min |
>
> FunASR 中文準確度約為 Whisper API 的 3 倍，且完全免費，**批次標註建議優先採用，省掉 STT API 費用**。
>
> **場景偵測升級：TransNetV2 取代 PySceneDetect**
> TransNetV2（神經網路）F1 達 87%，優於 PySceneDetect 的 65.5%，硬切與轉場均佳。
>
> **向量搜尋架構：pgvector HNSW + BM25 混合**
> - 向量搜尋找語意相似片段（`text-embedding-3-small`）
> - BM25 補強人名/地名精確比對（人名「賴清德」等向量搜尋容易漏）
> - 建議套件：ParadeDB pg_search（原生 Postgres BM25）
>
> **費用估算（更新）**：1 小時素材 ≈ 200 鏡頭 × $0.015/鏡頭（GPT-4o 影格）+ Whisper $0.36/hr ≈ **$3~5/hr**；FunASR 替換後降至 **$3/hr**（僅 GPT-4o 視覺費用）。

### 線 4 AI 輔助剪輯（長轉短 / 自動粗剪 / 多平台切版）
- **做什麼**：依文稿自動配片、出粗剪時間軸、長片轉多平台短影音。
- **怎麼做**：L2 腳本+選片配對 → L3 產 **Premiere 可開的時間軸** → L4 自動字幕＋多平台切版（橫↔直）。AI 出草稿、**人精修守品質**。
- **可行性**：🟡 中（XML 組裝可行；配片準確度需用自家資料調校；吸睛剪輯仍需人當導演）。
- **見效**：省時＋增量主力，接在搜片基礎成熟後做。

> 📎 **來源：[[研究彙整-國內外參考資料]]**
>
> **格式澄清：Premiere 匯入格式為 XMEML，非 FCPXML**
> 根目錄標籤為 `<xmeml version="4">`，NAS 路徑格式：`file://NAS-SERVER/ShareName/path/clip.mp4`
>
> **推薦工具**
> | 工具 | 用途 | 費用 |
> |---|---|---|
> | **auto-editor** | 自動剪靜音段 → Premiere XML，一行指令 | 免費 |
> | **OpenTimelineIO** | Python 程式化組裝時間軸（依腳本/選片） | 免費 |
> | Eddie AI | AI 粗剪雲端服務，Premiere XML 輸出，品質最佳 | $29/月起 |
>
> ```bash
> # auto-editor 最快入門（免費）
> pip install auto-editor
> auto-editor interview.mp4 --export premiere --margin 0.3s
> # → 輸出 interview_ALTERED.xml，Premiere File > Import 直接匯入
> ```

## 4. 技術架構（一張圖）
```
┌─ 地端（NAS／內網／各人電腦）─────────────┐
│  原始影片儲存 · 切鏡頭/抽音訊/抽縮圖        │
│  向量庫＋中繼資料 · 搜尋頁 · Premiere 剪輯   │
│  直播 OBS 疊字幕                          │
└───────────────┬──────────────────────────┘
                │ 只送輕量衍生物(音訊/影格/直播音流)
                ▼
┌─ 雲端（OpenAI）──────────────────────────┐
│  GPT-4o 看畫面/標籤 · embedding 向量      │
│  (直播)串流 STT＋翻譯                     │
└──────────────────────────────────────────┘
```
- **批次線**（庫管理）：事後處理，省成本（一鏡一影格）；STT 優先用本地 FunASR。
- **串流線**（直播字幕）：即時處理，重點在低延遲；用 gpt-realtime-whisper。

## 5b. 品質與準確率把關
- AI 只做「組裝苦力」，**不直接 render 成片**，交出可編輯時間軸；事實／法務／當事人／節奏／配樂版權 → **必經人工把關**。
- **準確率怎麼驗證／校正**（回答會議「怎麼信任 AI 選的題材與生成文字」）：上線前用自家素材調校、抽樣人工核對，達標才放行；編輯每次的修正回收成回饋料，讓選片／腳本準確率持續變準。

## 6. 分階段時程與里程碑
| 階段 | 工作 | 里程碑 |
|---|---|---|
| **Phase 1（快速見效）** | 線1 直播中文字幕＋線2a **數據型固定欄目**（首推：天氣/油價/賽事比分，可排程）＋線2b 舊素材模板化＋啟動線3 試點 | 直播有即時字幕；數據欄目可自動出片；舊素材能批次出片；試點搜片可 demo |
| **Phase 2（打基礎）** | 線3 搜片正式化＋線1 加英文翻譯字幕 | 編輯日常用搜片找素材；中英雙字幕直播 |
| **Phase 3（自動化剪輯）** | 線4 L2→L3→L4 | AI 出粗剪/多平台切版草稿，編輯精修出片 |
- 並行推：硬體升級盤點與採購、成立專案小組與窗口（會議已定）。
- **負責人與期程（預估，待拍板）**：Phase 1 由 AI 窗口（白天班同仁）＋AI 小組主導，估 2~4 週做出可展示成果；Phase 2 估 1~2 個月；Phase 3 估 2~3 個月起，視前階段成熟度。各項正式 owner 待專案小組成立後指派。

## 7. 暫緩／觀望（先不做，但要記著）
- ⏸️ **文字直接生成影片**：最難、品質未到、純 AI 畫面觀眾會關掉 → 觀望業界。
- ⏸️ **上雲隱私 ＋ AI 素材版權/真實性**：現階段暫緩；正式上線涉敏感素材前**務必回頭處理**。
- ⏸️ **AI 串接報社影音庫當生成素材來源**：成本高，先研究可行性與成本再提報。
