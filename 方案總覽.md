## 一、要解決什麼
影音新聞媒體，記者現場素材回傳，編輯**挑文案、選片、剪短影音**太慢。
導入 AI 達成：① 縮短產製時間 ② 大幅增加產片量 ③ **品質不能降**。

## 二、整體方案一句話
**素材一進庫就自動標註 → 編輯用一句國語就找到片段 → AI 出粗剪＋字幕＋多平台切版草稿 → 編輯精修出片。**

## 三、整套流程（全貌）
```
記者素材 → NAS(本地原檔)
   │
[標註] 切鏡頭 + Whisper逐字稿 + GPT-4o看畫面 + 自動標籤   ← 階段1
   │
[索引] 逐字稿+畫面描述 → 向量，存本地
   │
[搜尋] 打一句國語 → 片段清單(縮圖+來源檔+時間碼)        ← 階段1（試點驗證中）
   │
[輔助剪輯] 腳本+選片配對 → 自動粗剪(Premiere時間軸)      ← 階段2
   │        → 自動字幕 → 多平台切版(16:9↔9:16) → 各平台文案
   │
[編輯精修] 修剪/節奏/調色/把關 → 出片                     ← 人留在迴路
```

## 四、已拍板的決策
| 項目 | 決定 |
|---|---|
| 部署 | 混合式：**原檔留本地 NAS、AI 辨識一定走雲端**（內網無 GPU，不走地端 GPU） |
| 雲端 | **OpenAI**（Whisper + GPT-4o + text-embedding-3） |
| 隱私 | **現階段先忽略、暫緩處理**（⚠️ 擱置非解決，正式上線涉敏感素材時須重檢） |
| 剪輯軟體 | **Premiere Pro**（用 XML/FCPXML 匯入粗剪；可搭內建 AI） |
| 維運 | 編輯半技術自管 → 要傻瓜化、低維運 |
| 語言 | 國語為主 |
| 平台 | YouTube / Reels / Shorts / IG / WeChat |
| 量體 | 新素材 ~30GB/日；試點先跑半個月內的一個專題 |
| 起手式 | 先做**搜尋試點**證明效果與費用，再往剪輯走 |

## 五、品質怎麼守
AI 只做「組裝苦力」，**不直接 render 成片**，交出可編輯時間軸；
事實/法務/當事人/節奏/配樂版權等 → **必經人工把關**。

## 六、兩大目標各靠哪一塊
- **省時** ← 腳本+選片配對(L2)、自動粗剪(L3)、自動字幕。
- **增量** ← **多平台自動切版+各平台文案(L4)**（不加人力倍增產出）。

## 七、新增工作線：直播即時字幕／翻譯（來自會議）
- 需求：官網中／英文直播目前無字幕，要**即時字幕＋中→英翻譯**；會議認定**最快見效**。
- 技術獨立於影音庫，屬「**即時串流**」（streaming STT，非批次），可平行於試點推進。
- 最省力落地：先做中文即時字幕（串流 STT＋OBS 瀏覽器來源疊字幕）→ 再加英文翻譯；同步錄下事後校對。
- 詳見 [[專案彙整-會議與討論對照]] 第三節。

## 八、技術線釐清：兩條 GPU 別混
- **雲端**＝AI 標註／辨識（已拍板上雲）。
- **地端 GPU**＝剪輯／轉檔／回損（會議痛點，顯卡 3060→擬升 5070，省不掉）。
- 兩者不衝突、並存。

## 九、目前最大的未決點
1. 一支典型短影音的**結構/片長/旁白來源/配音方式**（決定階段2自動化怎麼設計）。
2. 各平台版本目前是「人工各剪一版」還是「一版改尺寸」（決定切版省時幅度）。

> 已收斂：AI 辨識上雲已拍板；隱私議題**暫緩**（先忽略，未來上線涉敏感素材再重檢）。
