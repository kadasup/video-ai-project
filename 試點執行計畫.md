## 1. 試點要回答的問題

1. AI 標出來的逐字稿 / 畫面描述，**準不準**（國語）？
2. 編輯**打一句話找片段**，能不能比現在快、且找得到對的？
3. 一批素材的**雲端費用**大概多少（推估每小時成本）？

跑完這三題有答案，就能決定要不要擴大、或日後轉地端 GPU。

## 2. 系統流程（資料怎麼流）

```
[專題素材資料夾] (放本地)
        │
   ① 預處理（本地，免費，不上雲）
        │  ffmpeg 抽音訊 (16k wav/mp3)
        │  PySceneDetect 偵測鏡頭切點 → 切成「片段 clip」
        │  每個片段抽 1 張代表影格 (縮圖)
        ▼
   ② 雲端標註（OpenAI）
        │  音訊 → Whisper → 逐字稿 (含時間碼)
        │  代表影格 → GPT-4o → 畫面描述 + 讀畫面文字(OCR) + 自動標籤
        ▼
   ③ 建索引（本地）
        │  每個片段：逐字稿+畫面描述 → text-embedding-3 → 向量
        │  存進 SQLite / Postgres(pgvector)
        ▼
   ④ 搜尋介面（本地內網）
        │  輸入國語 → 向量比對 → 回傳排序片段
        └─ 結果：縮圖 + 來源檔名 + 時間碼(可直接跳到 Premiere)
```

> 省錢關鍵：① 全在本地做、不花錢；只有 ② 上雲。且「一個鏡頭只送一張代表影格」而非整段影片。

---

## 3. 技術選型（OpenAI 綁定）

| 環節 | 用什麼 | 備註 |
|---|---|---|
| 抽音訊 / 切鏡頭 / 抽影格 | ffmpeg + PySceneDetect | 本地，CPU 即可 |
| 語音轉文字 | `whisper-1`（或更新的 `gpt-4o-transcribe`） | 國語佳；輸出含時間碼 |
| 畫面理解 + OCR + 標籤 | `gpt-4o`（vision） | 一次呼叫同時做「描述畫面＋讀字幕招牌＋給標籤」 |
| 語意向量 | `text-embedding-3-small`（試點夠用） | 便宜；要更準可換 large |
| 索引/搜尋 | SQLite + 向量欄位（試點輕量）→ 之後可升 pgvector | 單機、免維運 |
| 搜尋 UI | Next.js 單頁（內網） | 沿用既有技術棧 |

## 4. 片段資料結構（每個 clip 一筆）

```
clip_id          片段編號
source_file      來源原始檔名（編輯要回去 Premiere 找）
start_tc / end_tc 起訖時間碼
thumbnail        代表影格縮圖路徑
transcript       這段的逐字稿
visual_desc      GPT-4o 畫面描述
ocr_text         畫面上的文字（字幕/招牌/看板）
tags             自動標籤（地點/主題/物件/動作）
embedding        語意向量（供搜尋）
```

---

## 5. 搜尋體驗（試點長相）

- 一個搜尋框，打國語：例如「夜市人潮的空景」「記者站在路口講話的畫面」。
- 回傳排序卡片：**縮圖 + 來源檔名 + 時間碼 + 一句描述**。
- 點卡片 → 顯示「在哪個原始檔的第幾分幾秒」，編輯回 Premiere 直接跳該點。
- （進階，視時間）關鍵字篩選：日期、來源檔、標籤。

## 6. 成功指標（要量的數字）

| 指標 | 怎麼量 |
|---|---|
| ⏱️ 找片時間 | 同一個需求，AI 搜 vs 人工翻，各計時 |
| 🎯 命中率 | 前 5 筆結果裡有沒有「真的能用」的片段 |
| ✍️ 轉稿/描述準確度 | 抽樣人工核對國語逐字稿與畫面描述 |
| 💰 費用 | 跑完這批的 OpenAI 帳單 ÷ 素材總時長＝每小時成本 |

## 7. 費用量級（粗估，待試點實測校正）

以「一批約數小時素材」估算（僅供心理準備，非報價）：
- Whisper 轉稿：以分鐘計費，數小時素材通常僅數美元。
- GPT-4o 看影格：依代表影格張數而定（1 鏡頭 1 張），數百～上千張多在十美元級距內。
- Embedding：可忽略。
- **單一小專題試點，整體多落在「個位數～數十美元」量級。** 實際數字以試點帳單為準。
