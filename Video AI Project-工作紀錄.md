## 四條工作線現況

### 線 1 ⭐ 直播即時字幕
**狀態**：🟡 待開工（技術方案已定，可立即動手）

| 項目 | 內容 |
|---|---|
| 目標 | 官網直播加即時中文字幕，之後擴英文翻譯 |
| 推薦工具 | `gpt-realtime-whisper`（$0.017/min）+ OBS Browser Source |
| 已定技術 | WebSocket 串流、semantic VAD 斷句、`language:"zh"` 熱詞表 |
| 備援方案 | WhisperLive 自架（免費，PoC 測試用） |
| 已知限制 | Session > 9 分鐘延遲增加 → 每 8 分鐘重啟作 workaround |
| 實際延遲 | 1,660~1,860ms（非官稱 232ms） |
| **下一步** | **我寫好 PyAudio→字幕腳本，你接一場直播測試** |

> ⚠️ `gpt-4o-transcribe` 不可用於現場：真實會議環境 CER 31.40%（每 100 字錯 31 個）

---

### 線 2 ⭐ 模板化快速產製
**狀態**：🟡 待開工（最小切片可立即 demo）

| 項目 | 內容 |
|---|---|
| 目標 | 新聞稿/數據 → 自動出片，衝日產量 |
| 優先做 | **2a 數據型欄目**（天氣/油價/賽事），可排程無人值守 |
| TTS 選型 | Azure `zh-TW-HsiaoChenNeural`（台灣腔最佳，500K 字/月免費）；PoC 用 `edge-tts` 免費 |
| 模板工具 | Remotion（數據動畫）/ ffmpeg（字幕覆疊）；❌ After Effects 不可用 |
| **下一步** | **最小切片 demo**：新聞稿 → GPT-4o mini 腳本 → edge-tts → ffmpeg → mp4 |
| 待確認 | 短影音標準結構（片長、原聲或 TTS、平台版本數） |

---

### 線 3 影音庫標註 + 語意搜片
**狀態**：🔴 等素材路徑（機器與技術棧均已備妥）

| 項目 | 內容 |
|---|---|
| 目標 | 打一句國語找到片段，含自動預篩（濾掉糊晃/空拍） |
| PoC 主機 | 本機 i7-8700/32GB，WSL2，資料放 D:/H: |
| STT | FunASR Paraformer-zh（本地免費，CER 1.64%；比 Whisper API 準 3 倍） |
| 場景偵測 | TransNetV2（F1 87%，取代 PySceneDetect 65.5%） |
| 向量搜尋 | pgvector HNSW + BM25 混合（人名地名精確比對） |
| 視覺標註 | GPT-4o（$0.015/鏡頭，1hr 素材約 $3） |
| **卡點** | **需提供：試點素材資料夾路徑 + 大概時長/GB** |
| 費用估算 | 50~100 小時試點素材 ≈ $150~300 一次性 |

---

### 線 4 AI 輔助剪輯
**狀態**：⏳ 等線3 地基成熟後接續

| 項目 | 內容 |
|---|---|
| 目標 | 腳本+選片 → Premiere 時間軸草稿，人精修出片 |
| XML 格式 | Premiere 用 **XMEML**（非 FCPXML） |
| 推薦工具 | OpenTimelineIO（Python 生成時間軸）+ auto-editor（靜音段自動剪） |
| 付費選項 | Eddie AI（$29/月，AI 粗剪品質最佳） |
| 快速入門 | `auto-editor input.mp4 --export premiere --margin 0.3s` |
| **下一步** | 待線3 試點完成、搜片穩定後啟動 |
