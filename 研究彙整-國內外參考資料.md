## 一、線 1：直播即時字幕（Streaming STT）

### 核心結論：gpt-realtime-whisper 是現階段最佳平衡點

| 方案 | 費用 | 延遲 | 中文準確度 | 適合場景 |
|---|---|---|---|---|
| **gpt-realtime-whisper** ✅ | $0.017/min | 1~2s（實測） | 佳（Whisper large-v3 基底） | 直播正式上線 |
| WhisperLive（自架） | 免費 | 1~3s | 佳（本地 large-v2/v3） | PoC 測試 |
| Deepgram Nova-3 | $0.0048/min | <1s（官稱） | 中（2026-03 才加中文） | 英文或低成本場景 |
| gpt-4o-transcribe | $0.006/min | 非串流 | ⚠️ **危險**（見下） | **不建議用於現場** |

**⚠️ gpt-4o-transcribe 真實中文準確度警告**

多份學術論文（WenetSpeech benchmark）實測：
- 乾淨錄音（AISHELL-1）：CER 3.52% → 看起來很好
- **真實會議/現場環境（WenetSpeech Meeting）：CER 31.40%**
- 換算：每 100 個字就有 31 個字認錯 → **新聞直播不可接受**

對照：Whisper large-v3 在同場景 CER 18.87%（比 gpt-4o-transcribe 好 40%），FunASR/Paraformer 1.64%（最準）。

### 技術細節（實作關鍵）

**API 端點與格式**
```
wss://api.openai.com/v1/realtime?model=gpt-realtime-whisper
音訊格式：PCM16，24kHz（非 16kHz），mono，base64 編碼
```

**VAD 設定**
- `server_vad`：能量偵測，可調 threshold/silence_duration_ms
- `semantic_vad`：ML 分類器（eagerness: low/medium/high），斷句更自然 → **推薦**

**Python 串接範本（直播音訊 → 字幕）**
```python
import asyncio, websockets, json, base64, pyaudio

async def live_caption(api_key: str, language: str = "zh"):
    uri = "wss://api.openai.com/v1/realtime?model=gpt-realtime-whisper"
    headers = {"Authorization": f"Bearer {api_key}",
               "OpenAI-Beta": "realtime=v1"}
    
    async with websockets.connect(uri, extra_headers=headers) as ws:
        # 初始化 session：指定語言 + semantic VAD
        await ws.send(json.dumps({
            "type": "session.update",
            "session": {
                "modalities": ["text"],
                "input_audio_format": "pcm16",
                "input_audio_transcription": {
                    "model": "whisper-1",
                    "language": language        # "zh" 提升中文準確度
                },
                "turn_detection": {
                    "type": "semantic_vad",
                    "eagerness": "medium"       # 不要太急著斷句
                }
            }
        }))
        
        # 收音 + 送串流（PyAudio，24kHz）
        p = pyaudio.PyAudio()
        stream = p.open(format=pyaudio.paInt16, channels=1, rate=24000,
                        input=True, frames_per_buffer=4096)
        
        async def send_audio():
            while True:
                chunk = stream.read(4096, exception_on_overflow=False)
                await ws.send(json.dumps({
                    "type": "input_audio_buffer.append",
                    "audio": base64.b64encode(chunk).decode()
                }))
                await asyncio.sleep(0)
        
        async def recv_captions():
            async for msg in ws:
                data = json.loads(msg)
                if data.get("type") == "conversation.item.input_audio_transcription.completed":
                    print(data["transcript"])   # → 送 OBS 字幕來源
        
        await asyncio.gather(send_audio(), recv_captions())
```

**OBS 字幕整合**：用本地 WebSocket server（`websockets` library）接收字幕 → OBS Browser Source 連接 `ws://localhost:8765` 顯示字幕覆疊。

**熱詞（新聞專名）注入**
```python
"input_audio_transcription": {
    "model": "whisper-1",
    "language": "zh",
    "prompt": "以下是台灣新聞播報，涉及人名、地名、機構：賴清德、立法院、國防部"
}
```
→ 維護一份「新聞熱詞表」定期更新 prompt，可顯著提升專名識別率。

**已知問題**
- Session 超過 9 分鐘後延遲可能增加到 6~8s（OpenAI 已知 bug） → 每 8 分鐘重啟 session 作為 workaround
- 實際延遲：OpenAI 宣稱 232ms，獨立測試（webrtcHacks）玻璃到玻璃為 1,660~1,860ms → 規劃字幕 delay 約 2s

### 向量搜尋架構：pgvector + BM25 混合

```sql
-- HNSW 索引（高效近似搜尋）
CREATE INDEX ON clips USING hnsw (embedding vector_cosine_ops)
    WITH (m = 16, ef_construction = 200);

-- 搜尋時設定 ef_search
SET hnsw.ef_search = 100;

-- 混合搜尋（向量 + BM25）- 用 ParadeDB pg_search
SELECT c.clip_id, c.thumbnail, c.transcript, c.start_tc, c.source_file,
       (0.7 * vector_score + 0.3 * bm25_score) AS combined_score
FROM clips c
ORDER BY combined_score DESC
LIMIT 10;
```

**為何需要 BM25**：人名「賴清德」、地名「信義區」用向量搜尋常找不到（因為 embedding 找語意相似，不找字面匹配）。BM25 補強精確關鍵字搜尋，RRF 合併兩路結果。

### GPT-4o 視覺標註省成本

```python
# 每個鏡頭只送 1 張代表影格（而非整段影片）
import base64, openai

def annotate_keyframe(frame_path: str, client: openai.OpenAI) -> dict:
    with open(frame_path, "rb") as f:
        b64 = base64.b64encode(f.read()).decode()
    
    response = client.chat.completions.create(
        model="gpt-4o",                  # vision 功能
        messages=[{
            "role": "user",
            "content": [
                {"type": "image_url",
                 "image_url": {"url": f"data:image/jpeg;base64,{b64}",
                               "detail": "low"}},  # low detail 省 token
                {"type": "text",
                 "text": """分析這張新聞影像，以繁體中文輸出 JSON：
{
  "scene_type": "戶外/室內/記者會/採訪/現場",
  "location": "推測地點",
  "subject": ["畫面中的人物或主體"],
  "action": "正在發生的事情",
  "ocr_text": ["畫面上的文字"],
  "tags": ["新聞標籤，3~8個"],
  "news_value": "high/medium/low",
  "notes": "其他補充"
}"""}
            ]
        }],
        max_tokens=300
    )
    return response.choices[0].message.content
```

**費用估算**：1 個鏡頭 1 張影格 → 約 $0.01~0.02/鏡頭，1 小時素材（約 200 鏡頭）≈ $2~4。加 Whisper 轉稿 $0.36/hr → **1 小時素材標註成本約 $3~5**（遠低於人工 30 分鐘的時間成本）。

## 四、線 4：AI 輔助剪輯（Premiere XML 自動化）

### Premiere XML 格式澄清

Premiere 使用 **XMEML**（XML 格式），非 FCPXML：
```xml
<?xml version="1.0" encoding="UTF-8"?>
<xmeml version="4">
  <sequence>
    <name>RoughCut</name>
    <timebase>25</timebase>
    <media>
      <video>
        <track>
          <clipitem id="clip1">
            <name>shot_001</name>
            <in>0</in>
            <out>125</out>   <!-- 5秒 × 25fps -->
            <file id="f1">
              <name>shot_001.mp4</name>
              <!-- NAS 路徑格式 -->
              <pathurl>file://NAS-SERVER/MediaShare/News/shot_001.mp4</pathurl>
            </file>
          </clipitem>
        </track>
      </video>
    </media>
  </sequence>
</xmeml>
```

### OpenTimelineIO 生成 Premiere XML

```python
pip install opentimelineio otio-cmx3600-adapter
```

```python
import opentimelineio as otio

def create_rough_cut(clips: list[dict], fps: float = 25.0, 
                     output_path: str = "rough_cut.xml"):
    """
    clips: [{"file": "file://NAS/share/clip.mp4", "start": 0, "end": 125}, ...]
    """
    timeline = otio.schema.Timeline(name="AI_RoughCut")
    video_track = otio.schema.Track(name="V1", kind=otio.schema.TrackKind.Video)
    audio_track = otio.schema.Track(name="A1", kind=otio.schema.TrackKind.Audio)
    timeline.tracks.extend([video_track, audio_track])
    
    for i, c in enumerate(clips):
        clip = otio.schema.Clip(
            name=f"clip_{i:03d}",
            media_reference=otio.schema.ExternalReference(
                target_url=c["file"]   # NAS UNC 路徑
            ),
            source_range=otio.opentime.TimeRange(
                start_time=otio.opentime.RationalTime(c["start"], fps),
                duration=otio.opentime.RationalTime(c["end"] - c["start"], fps),
            ),
        )
        video_track.append(clip)
        audio_track.append(clip.clone())
    
    # 輸出 XMEML（Premiere 可直接匯入）
    otio.adapters.write_to_file(timeline, output_path, adapter_name="fcp_xml")
    print(f"✅ 輸出：{output_path}（Premiere File > Import 匯入）")
```

**NAS 路徑格式**：
- UNC 格式：`file://NAS-SERVER/ShareName/path/clip.mp4`
- 或掛載磁碟：`file://localhost/Z:/path/clip.mp4`

### 端到端流程：Whisper → EDL → Premiere

```python
# 完整流程：音訊 → 逐字稿 → 依關鍵詞選片 → Premiere XML

from faster_whisper import WhisperModel
import opentimelineio as otio

# Step 1: 本地 STT（或換 FunASR）
model = WhisperModel("large-v2", device="cuda", compute_type="float16")
segments, info = model.transcribe(
    "interview.wav", language="zh",
    initial_prompt="以下是台灣新聞採訪，請以繁體中文輸出",
    vad_filter=True
)

# Step 2: 依關鍵詞篩選片段（或用 GPT-4o 智能選片）
keywords = ["政策", "記者會", "聲明"]
selected = [
    {"start": seg.start, "end": seg.end, "text": seg.text}
    for seg in segments
    if any(k in seg.text for k in keywords)
]

# Step 3: 輸出 Premiere XML
clips = [{"file": f"file://NAS/News/interview.mp4",
           "start": int(c["start"] * 25),
           "end": int(c["end"] * 25)}
         for c in selected]
create_rough_cut(clips, output_path="rough_cut.xml")
```
